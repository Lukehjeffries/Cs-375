This week I began researching the backend development for our course planner in order to determine how the system should manage and process course data. I focused on understanding how course information could be structured in a backend environment, including how classes, sections, credits, and scheduling information would be represented in the system. I also looked into how the backend would handle user actions such as selecting courses, building a schedule, and validating constraints such as course conflicts or maximum enrollment.

Part of this research involved thinking about the overall system architecture and how the backend would interact with the frontend interface. I considered how a database or structured data storage could be used to hold course catalog information and how the application logic would retrieve and update that data when a user interacts with the planner. I also explored possible data structures that could represent courses and schedules in code so that the system can efficiently check for conflicts or changes in a student’s schedule.

For the upcoming work this week, my goal is to begin implementing a rough backend prototype. This will involve creating an initial code structure that represents course objects and basic scheduling functionality. The goal of this early implementation is not to produce a finalized system, but rather to create a functional starting point that demonstrates how courses could be stored, selected, and organized within the planner.

Developing this early prototype will allow our group to test ideas and identify potential technical challenges before moving further into development. It will also give the team a concrete framework that we can build upon in later sprints as we expand the system’s functionality and integrate it with other parts of the project.
